Introduction to Rokt Public API

Why Rokt Public API

Rokt Public API is a solution for Rokt clients to retrieve their performance data from the Rokt platform. It's an HTTP-based API that Rokt clients can use to programmatically query and integrate Rokt data into their business applications. It allows Rokt clients to build their own reports and dashboards based on the data retrieved from the Rokt platform.

Integrating with the Rokt Public API makes data available on a timely basis to help clients make better budgeting and performance optimisation decisions.

Consumers

The Rokt Public API is designed for developers. As an existing partner of Rokt, with sufficient technical knowledge you are able to invoke various endpoints of the Rokt Public API and integrate the retrieved data into your own applications.

HTTP Based

The Rokt Public API is HTTP-based and works with any language that supports making HTTP requests. Almost all requests are passed to the https://api.rokt.com host URL.

All data transfers are based on HTTP/1.1, and all endpoints require HTTPS. It is organized around REST. The API accepts JSON-encoded request bodies, returns JSON-encoded responses, and uses standard HTTP response codes, authentication, and verbs.

Authentication

The Rokt Public API uses access tokens to authenticate requests. Authentication to the API is performed via OAuth 2.0. You'll be provided with client credentials that can be used to call endpoints on the API. You do not need to provide a password.

API requests without authentication will fail. From the Rokt Public API, you will only be able to retrieve data from the accounts that your user credentials have access to.

Errors

Rokt Public API uses conventional HTTP response codes to indicate the success or failure of an API request. In general:

  • Codes in the 2xx range means success.
  • Codes in the 4xx range means an error that exists in the information provided (e.g., a required parameter was omitted, a date range is incorrect, etc.)
  • Codes in the 5xx range means an error with Rokt's servers (these are infrequent).
