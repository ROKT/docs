Sandbox Integration

This scenario is an extensions to other integration examples and explains how to pass a configuration flag to test against a sandbox environment. A sandbox environment is a one to one production counterpart which does not generate referrals. Except that it follows a normal offers biding and matching process against production configuration. As a result it can be used for acceptance testing before deploying to production.

The integration follows exactly the same steps as in previous examples but additional sandbox flag is required to be passed to the initialisation function.

See Rokt.init for more details on the method

rokt.onLoaded(function(rokt) {
  rokt.init({
    sandbox: true,
  });

  // Integration steps as per other examples
});

Once sandbox flag is set you should see Rokt's solution executing API calls against https://public-api-demo.rokt.com instead of production https://public-api.rokt.com which ensures sandboxing works correctly.

The sandbox flag should only be used in testing environments!
